🦞 OpenClaw Setup Guide

Hướng dẫn đầy đủ cài đặt OpenClaw + 9router Bypass API + Telegram Bot trên macOS

OpenClaw v2026.4.5 9router v0.3.82 macOS Node.js ≥ 22.12
Node.js ≥ 22.12.0
npm ≥ 11.x
macOS Monterey+
1
Cài OpenClaw
▾
Chạy installer script
bash
# Cài OpenClaw global qua curl
curl -fsSL https://openclaw.ai/install.sh | bash

# Verify cài đặt thành công
openclaw --version
# → OpenClaw 2026.4.5 (3e72c03)
2
Khởi động Gateway
▾
bash
# Khởi động gateway daemon (tự restart khi reboot)
openclaw gateway start

# Kiểm tra trạng thái
openclaw status
ℹ️ Gateway chạy tại ws://127.0.0.1:18789 · Control UI: http://127.0.0.1:18789
3
Kết nối Telegram Bot
▾
3.1 Tạo bot từ BotFather
  1. Vào Telegram → tìm @BotFather
  2. Gửi /newbot → đặt tên → lấy Bot Token
3.2 Đấu nối bot vào OpenClaw
bash
openclaw channels add --channel telegram --token <BOT_TOKEN>

# Verify
openclaw channels status --probe
# → Telegram default: enabled, configured, running, works ✅
3.3 Pair tài khoản Telegram của bạn
  1. Vào bot Telegram → gửi /start
  2. Bot trả về Pairing Code (VD: F2F7D3JA)
  3. Chạy lệnh approve trên máy:
bash
openclaw pairing approve telegram <PAIRING_CODE>
# → Approved telegram sender <user_id>
4
Cài 9router — API Bypass Proxy
▾
4.1 Cài và khởi động
bash
npm install -g 9router

# Chạy background (system tray)
9router --tray
# → 🚀 9router v0.3.82
# → Server: http://localhost:20128
🌐 Dashboard: http://localhost:20128/dashboard
4.2 Kết nối API vào 9router dashboard
  1. Mở dashboard → Providers → chọn provider (VD: OpenAI Codex)
  2. Click Add account → nhập API key → Save
  3. Vào Available Models → note các model ID (VD: cx/gpt-5.4)
⚠️ QUAN TRỌNG: Model của 9router dùng prefix riêng như cx/gpt-5.4, KHÔNG phải gpt-4o chuẩn. Phải dùng đúng model ID này khi config OpenClaw.
5
Cấu hình OpenClaw dùng 9router
▾
5.1 Sửa ~/.openclaw/openclaw.json
json
{
  "gateway": {
    "mode": "local",
    "auth": { "mode": "token", "token": "<GATEWAY_TOKEN>" }
  },
  "channels": {
    "telegram": {
      "token": "<BOT_TOKEN>", "enabled": true,
      "accounts": { "default": { "token": "<BOT_TOKEN>" } },
      "botToken": "<BOT_TOKEN>"
    }
  },
  "agents": {
    "defaults": {
      "memorySearch": { "enabled": false },
      "model": { "primary": "9router/cx/gpt-5.4" }
    }
  },
  "models": {
    "mode": "merge",
    "providers": {
      "9router": {
        "baseUrl": "http://localhost:20128/v1",
        "apiKey": "sk-9router",
        "api": "openai-completions",
        "models": [
          { "id": "cx/gpt-5.4", "name": "cx/gpt-5.4" },
          { "id": "cx/gpt-5.3-codex", "name": "cx/gpt-5.3-codex" },
          { "id": "cx/gpt-5.1-codex-mini", "name": "cx/gpt-5.1-codex-mini" }
        ]
      }
    }
  }
}
💡 apiKey có thể là bất kỳ string — 9router không validate, tự dùng key đã nhập trong dashboard. GATEWAY_TOKEN lấy từ file openclaw.json gốc.
5.2 Thêm env vars vào LaunchAgent plist

File: ~/Library/LaunchAgents/ai.openclaw.gateway.plist — thêm vào trong block <EnvironmentVariables> trước </dict>:

xml
<key>OPENAI_API_KEY</key>
<string>sk-9router-bypass</string>
<key>OPENAI_BASE_URL</key>
<string>http://localhost:20128/v1</string>
5.3 Reload daemon
bash
# Unload daemon cũ
launchctl bootout gui/$(id -u)/ai.openclaw.gateway

# Load lại với config mới
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# Verify env vars đã load
launchctl print gui/$(id -u)/ai.openclaw.gateway | grep OPENAI
# → OPENAI_API_KEY => sk-9router-bypass
# → OPENAI_BASE_URL => http://localhost:20128/v1
5.4 Verify model config
bash
openclaw models status
# → Default: 9router/cx/gpt-5.4
# → 9router effective=models.json ✅

Troubleshooting
▾
LỗiNguyên nhânFix
401 Incorrect API key: sk-9router... OpenClaw gọi thẳng OpenAI thật, không qua 9router Kiểm tra OPENAI_BASE_URL trong plist đã reload chưa
404 No active credentials for provider: openai 9router nhận gpt-4o nhưng không có OpenAI key thật Đổi model sang cx/gpt-5.4 trong openclaw.json
Unknown model: cx/gpt-5.4 Custom provider chưa khai báo trong config Thêm models.providers.9router vào openclaw.json
pairing required 2 gateway instances chạy cùng lúc openclaw gateway stop → openclaw gateway start
Something went wrong Session cũ bị lỗi / model chưa set Gửi /new trong Telegram; kiểm tra openclaw models status
Bot im lặng sau reboot 9router chưa chạy lại Chạy: 9router --tray

Maintenance
▾
bash
# Dọn session lỗi
openclaw sessions cleanup \
  --store ~/.openclaw/agents/main/sessions/sessions.json \
  --enforce --fix-missing

# Kiểm tra sức khỏe
openclaw doctor

# Xem log realtime
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log

# Cập nhật
openclaw update
npm update -g 9router
